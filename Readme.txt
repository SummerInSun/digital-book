适用板子：JZ2440，确切来说是带有驱动的所有 linux 板，但是源码里面的 freetype.c 里面的 lcd 屏分辨率是 480*272 的，没有做进一步的适应性改动，所以其他的屏需要改动 272 为使用的屏幕的 Y 方向尺寸

*******项目所用到的文件*******
freetype
触摸屏驱动
framebuffer
按键驱动

*******文件*******
1.顶层目录里面的 TB-xx.txt 是各种编码格式的三体小说文本文件
2.有几个 *test.c 的文件是在项目过程中的测试文件，从名字可以知道大概作用， main.c 是最终的测试文件
3.顶层目录里面的 system 文件是 list_head 结构体双向链表的操作函数，包括遍历，添加，删除。与之相关的头文件是 include/system/common_st.h 。include/system/common_config.h 里面是通用配置，包括电子书颜色，打印级别定义
4.input_select 目录里面是做用 select 方式输入的遗留，input 目录是最终的多线程输入。输入里面添加了按键输入
5.netprint 目录里面的是远程打印相关代码。里面的 client 与 client.c 分别是打印客户端的程序与文件，用在 PC 机上


*******与韦老师视频所写项目主要不同的地方*******
1.所有的结构体注册采用与内核相似的 list_head 双向链表完成，该链表嵌套在要注册的结构体里面，主要操作函数在 system/common_st.c 里面
2.加了按键输入
3.网络打印是根据韦老师讲的思路自己独立完成的
4.其他还有一些小的改动，比如字体设置啦，触摸屏操作啦，还有稍稍改进了下输入设备注册，使得只要有一个输入设备注册成功就可以运行程序，不用全部都要注册成功。当然编码以及字体的注册也可以做这个改进


*******测试*******
	1.电子书测试：
	基本命令： exe <fontfile> <text> <size>
	例如：./test simfang.ttf ./TB-ansi.txt 16

	键盘操作：
	n	下翻
	u	上翻
	q	退出
	l	字体倾斜(仅freetype显示有效，不适用与 ansi 编码的文件)
	o	加粗字体
	e 	给字体加下划线
	r	恢复正常字体

	触摸屏：
	从中间往下滑	下翻
	从中间往上滑	上翻
	从中间往右滑	退出
	点击上边中间	上翻
	点击下边中间	下翻
	点击右边中间	退出

	按键：
	EINT0	下翻
	EINT1	上翻
	EINT11	退出
	
	2.网络打印测试：
	客户端：
	/* 仅仅对客户端来说，板子只受 net=0|1 console=0|1 控制 */
	debug		接收板子的打印信息到 PC 客户端
	closedebug	关闭板子的打印信息
	
	net=0|1		关闭/开启板子到远程客户端的打印
	console=0|1	关闭/开启板子到串口的打印
	debuglevel=<0-7>设置默认打印级别，大于等于此级别的都被打印
	
*******已知的 bug*******
1.freetype 显示小于 24 的字体会出现页面第一行显示缺失的情况
2.网络客户端控制执行完 debug 之后如果没有接收到打印数据再执行 closedebug 的时候会阻塞，直到有数据发过来为止才会退出 debug 状态
